## 线程池

> 池化思想：线程池、字符串常量池、数据库连接池

- 线程池的优势

  1. 手动创建线程对象
  2. 执行任务
  3. 执行完毕，释放线程对象

- 线程池的优点

  1. 提高线程的利用率
  2. 提高程序的响应速度
  3. 便于统一管理线程对象
  4. 可以控制最大的并发数

### 线程池执行流程

1. 核心线程未满，进入核心线程执行
2. 核心线程满，阻塞队列未满，进入阻塞队列
3. 阻塞队列满，创建非核心线程并执行
4. 非核心线程也满，执行拒绝策略

- 为什么要先进阻塞队列、再创建非核心线程池

  以银行柜台为例，人多时先排队（进队列），而不是（开柜台）。同理开线程的资源消耗更大。
